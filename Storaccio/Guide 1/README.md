# Guide 1 - Real-Time Systems

Collection of exercises in C from Guide 1 of **Real-Time Systems**.

## Content

- `exercise1.c`: basic calculator (addition, subtraction, multiplication and division).
- `exercise2.c`: word counter in a line of text.
- `exercise3.c`: sorting integers with Bubble Sort.
- `exercise4.c`: traffic light simulation with non-blocking pedestrian button detection.
- `exercise5.c`: current clock or stopwatch using `clock_gettime`.

## Requirements
- GCC (recommended `gcc-12`)
- Linux (specially for `exercise4.c` for `select`, `unistd`, `STDIN_FILENO`)

## Compile and Execute

From folder `Guide 1`:

```bash
gcc-12 exercise1.c -o exercise1 && ./exercise1
gcc-12 exercise2.c -o exercise2 && ./exercise2
gcc-12 exercise3.c -o exercise3 && ./exercise3
gcc-12 exercise4.c -o exercise4 && ./exercise4
gcc-12 exercise5.c -o exercise5 && ./exercise5
```

If your environment requires it for POSIX-time functions, you can compile `exercise5.c` with:

```bash
gcc-12 exercise5.c -o exercise5 -lrt
```

## What each exercise does

### 1) Calculator (`exercise1.c`)

It asks for a choice from 1 to 4 and two floating-point numbers, then executes:

1. Sum
2. Substract
3. Multiplication
4. Division (with division by zero validation)

### 2) Word counter (`exercise2.c`)

Read a line of text and count how many words it contains, ignoring spaces, tabs, and line breaks.

### 3) Bubble Sort (`exercise3.c`)

Request `n` integers, display the entered array, and then sort it in ascending order using Bubble Sort.

### 4) Traffic light with pedestrian button (`exercise4.c`)

It simulates states `GREEN -> YELLOW -> RED` in a loop and detects a pressed key without blocking the main execution (`select`).

### 5) Watch / Stopwatch (`exercise5.c`)

Allows you to choose between:

- **Actual time** of the system.
- Continuous **Stopwatch** in `hh:mm:ss` format.

## Notes

- Some program messages are in English and others in Spanish; this does not affect functionality.
- There are binaries (`exercise 4`, `exercise 5`) in the folder that can be regenerated by compiling again.
